Judging Checklist
-----------------

Use this checklist when evaluating the repo for the hackathon criteria:

- **GitHub usage**: issues, branches, PRs, workflows, meaningful commits
- **Copilot visibility**: See `docs/COPILOT_USAGE.md` and `git log` for `copilot/seed` branch
- **Code quality**: linting, simple unit tests, clear module boundaries
- **Docs**: clear README, CONTRIBUTING, templates, and examples
